装 CCS Theia + 导入 MSPM0 SDK

TI 官方 IDE。用 SysConfig 图形化配引脚/时钟/外设，不用手写初始化代码
为什么用 CCS Theia？
MSPM0 系列是 TI 的芯片，TI 官方 IDE 就是 CCS（Code Composer Studio）。Theia 是最新版，基于 VS Code 内核，界面你熟悉。

最重要的功能：SysConfig——图形化配引脚。你点一点选一选，它自动生成所有外设初始化代码。不用翻 1000 页参考手册手写寄存器。

DriverLib： TI 的库函数，类似 STM32 的 HAL 库。你用 DL_ADC12_startConversion() 而不是操作 ADC12 的寄存器。
安装步骤
1
下载 CCS Theia
去 TI 官网 (ti.com) 搜索 "CCS Theia"，下载 Windows 版安装包。安装路径用默认 C:\ti\ccs2051\。
2
下载 MSPM0 SDK
同样 TI 官网搜索 "MSPM0 SDK"，下载最新版（≥ 2.10）。安装时勾选 MSPM0G3507 支持。默认路径 C:\ti\mspm0_sdk_2_10_00_04\。
3
新建工程
CCS → File → New Project → 选 MSPM0G3507 → 选 "Empty Project with DriverLib" → 工程名 racecar_h。
4
打开 SysConfig 配引脚
工程里双击 .syscfg 文件 → 打开图形化配置界面。
SysConfig 怎么配（重要）
SysConfig 打开后左边是外设列表，右边是引脚图。你需要添加以下外设：
外设数量用途配什么引脚
ADC128 通道8 路灰度传感器PA0~PA7（或任意 8 个模拟引脚）
TIMx (PWM)2 通道左右电机 PWM 输出任意 GPIO（配成 TIM PWM 输出）
TIMx (Encoder)2 个左右编码器正交解码任意 GPIO（配成 TIM 编码器输入）
I2C1 个OLED + MPU6050PB2=SCL, PB3=SDA
UART1 个串口调试 / printf任意 UART TX/RX 引脚
GPIO3 个蜂鸣器 + LED + 按键任意数字引脚，配成输出（蜂鸣器/LED）和输入（按键）
SYSCLK—系统时钟 80MHz配 SYSPLL + 时钟分频
配完后点右上角 Save，SysConfig 自动生成 ti_driver_config.c 和 ti_driver_config.h。 你的 bsp 代码直接 #include "ti_driver_config.h"，调 DriverLib 函数。
SysConfig 配好后你的 bsp 层怎么写
/* bsp_adc.c — SysConfig 已经生成了 ADC 初始化代码，你只需要调 DriverLib 函数 */ #include "ti_driver_config.h" // SysConfig 自动生成的 #include "bsp_adc.h" void bsp_adc_init(void) { // SysConfig 自动生成了 ADC12_init()，在 main() 的 SystemInit 里会调。 // bsp_adc_init 只需要做附加配置（如果有的话）。 // 大部分情况下这个函数体是空的——SysConfig 已经帮你配好了。 } uint16_t bsp_adc_read_ch(uint8_t ch) { DL_ADC12_startConversion(ADC12_INST); // 启动单次转换 while (!DL_ADC12_isMemResultReady(ADC12_INST, ch)); // 等完成 return DL_ADC12_getMemResult(ADC12_INST, ch); // 读 0~4095 } /* 对比 STM32 HAL：HAL_ADC_Start() → HAL_ADC_PollForConversion() → HAL_ADC_GetValue() * 逻辑完全一样，只是函数名变了。DriverLib 前缀是 DL_ 而不是 HAL_ */
常见翻车
❌ SysConfig 配了但代码没 include → #include "ti_driver_config.h" 是必须的，否则所有外设都没初始化
❌ SDK 版本不匹配 → CCS 新建工程时选的 SDK 版本和安装的不一致。Project → Properties → 确认 SDK 路径
❌ SysConfig 改了没重新生成 → 每次改 .syscfg 后必须点 Save 触发重新生成，否则代码还是旧的
❌ 调试器选错 → 天猛星板载 DAP-Link，CCS 的 Debug Configuration 里选 CMSIS-DAP，不是 XDS110
❌ 下载提示 "Error connecting to target" → 板子没上电、杜邦线松了、或 Debug Configuration 里选错了调试器类型
