路径状态机：车怎么知道该往哪走？

你的车没有眼睛——它靠状态机告诉自己"我现在在哪一段、下一段要做什么"
要求 3 的四个状态
S0
A → C 对角线
IMU 航向锁定
白地无标记 · 陀螺仪+编码器
S1
C → B 弧线
巡黑线 · 降速+大P
有黑线可巡 · 灰度+陀螺仪
S2
B → D 对角线
IMU 航向锁定
白地无标记 · 陀螺仪+编码器
S3
D → A 弧线
巡黑线 · 降速+大P
有黑线可巡 · 灰度+陀螺仪
S0 → S1 → S2 → S3 → 圈数+1 → S0 ×4 → STOP
最关键的问题：怎么判断"到顶点"了？
主
编码器里程判断（所有状态通用）
每进入新状态时清零 distance。里程到了预设值（SEG_DIAG 或 SEG_ARC）就切状态。
if (distance >= seg_target) { state++; mod_encoder_reset(); }
备
灰度丢失判断（仅弧线段 S1/S3）
弧线顶点处黑线突然没了 → 灰度全白 10 次 → 确认到了顶点。对角线段 S0/S2 不适用（本来就全白）。
if (lost_count > 10) { state++; lost_count = 0; }
S0/S2（白地对角线）和 S1/S3（弧线黑线）策略完全不同
控制策略对比
  • S0 & S2（白地）：灰度读不到东西 → 方向环输入从 line_pos 切换为 (target_heading - actual_heading) → 陀螺仪全力锁定航向。这是最容易跑偏的两段
  • S1 & S3（弧线）：灰度能读到黑线偏移 → 方向环正常工作，但要降速 + 弧线专用大 KP（通常直道 KP 的 2~3 倍），防止离心力甩出去
  • S0 和 S2 的航向不同：进场后先原地转一圈，陀螺仪测出 A→C 和 B→D 的实际角度，写死到 HEADING_AC 和 HEADING_BD 常量
