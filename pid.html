PID 控制：怎么让车沿着黑线走？

两层 PID 套在一起：外层决定"往哪拐"，内层决定"跑多快"
为什么需要两层 PID？
一层不够吗？不够。

如果只有方向环：车知道偏右了要左转，但它不知道转多少"刚刚好"——可能转太少纠正不回来，也可能转太多又偏左了。
如果只有速度环：车能跑稳一个速度，但它不知道黑线在哪——直直地冲出赛道。

两层套一起：方向环算出"需要转多少"，把这个值加到两轮速度差上。速度环确保"转的同时速度不掉"。两个环各管各的，互相不打架。
串级 PID 结构
方向环（外环 · 位置式 PID）
输入 line_pos（灰度偏移 -100~+100）→ 输出 turn_amount（转向修正量）
偏右就往左打，偏左就往右打。就这一句话
▼ turn_amount 叠加到两轮速度上 ▼
速度环 ×2（内环 · 增量式 PID）
左轮目标 = base + turn  |  右轮目标 = base − turn
PID 算出实际 PWM → 编码器驱动 → 520 电机 → 编码器反馈 → 闭环
增量式 PID 公式
Δu = Kp(ek − ek−1) + Ki·ek + Kd(ek − 2ek−1 + ek−2)
uk = uk−1 + Δu  （当前输出 = 上次输出 + 增量）
为什么速度环用增量式而不是位置式？
位置式：u = Kp×e + Ki×Σe + Kd×(e-eprev)。积分项 Σe 一直累加，如果电机堵转了（error 一直很大），积分疯狂暴涨 → 即使后来解除堵转，输出已经飞了。这是积分饱和。

增量式：unew = uold + Δu。每次只变一点点。堵转时 Δu 有限，输出不会飞。车从直道切弧线时速度突变，增量式自然过渡，不会"窜"一下。

M0+ 算得过来：只存最近 3 次误差，一个周期就两行乘加。
三个参数一句话
Kp 比例
大了：车左右扭(S形)
小了：纠正太慢跑偏
作用：误差越大，纠偏越猛
Ki 积分
大了：速度过冲
小了：永远差一点(静差)
作用：消除稳态误差
Kd 微分
大了：电机嗡嗡抖
小了：急弯反应不过来
作用：预判趋势，抑制震荡
每段代码的作用解释
/* 方向环：输入 line_pos，输出 turn_amount */ int16_t pid_direction(float error, float kp, float kd) { static float last = 0; float out = kp * error + kd * (error - last); last = error; return (int16_t)out; } /* 效果： * line_pos = +30（偏右了）→ error = +30 * → kp * 30 = 正数 → turn 为正 → 左轮加速右轮减速 → 车左转纠正 ✓ * line_pos = -20（偏左了）→ error = -20 * → kp * (-20) = 负数 → turn 为负 → 左轮减速右轮加速 → 车右转纠正 ✓ * 为什么不用 Ki？方向环不需要消除静差——车不需要"精确停在黑线正中间"， * 黑线宽度 1.8cm 有容差。Ki 反而会在弧线段造成积分饱和 */
/* 速度环：输入 target & actual，输出 PWM */ uint16_t pid_speed(float target, float actual, uint8_t side) { static float e1[2]={0,0}, e2[2]={0,0}, u[2]={0,0}; float e = target - actual; float inc = SPD_KP*(e-e1[side]) + SPD_KI*e + SPD_KD*(e-2*e1[side]+e2[side]); u[side] += inc; if (u[side] > 1000) u[side] = 1000; if (u[side] < 0) u[side] = 0; e2[side] = e1[side]; e1[side] = e; return (uint16_t)u[side]; } /* 效果： * target = 600（目标速度），actual = 580（实测慢了一点） * → e = +20 → inc 为正 → u 增加 → PWM 加一点 → 加速到 600 ✓ * target = 600，actual = 630（实测快了一点） * → e = -20 → inc 为负 → u 减少 → PWM 减一点 → 减速到 600 ✓ * 限幅 0~1000：PWM 不会超出合法范围 */
常见翻车
❌ KP 太大 → 直道上车左右扭（S 形摆动）。不是车坏了，是纠正过度——方向环打方向太猛，每次都越过中线
❌ KP 太小 → 车慢慢偏出赛道。纠正力度不够，偏了回不来
❌ KI 过大 → 速度过冲：加油门冲上去、再松油回落、再冲……像坐过山车。增量式的 Ki 作用比位置式温和，但还是不能太大
❌ static 变量忘记用 side 区分左右 → e1[0]和e1[1]必须是两个独立的值。如果共用 → 左轮的 PID 会影响右轮的计算 → 两边互相干扰
❌ 限幅没写 → PWM 可能超出 1000 或变负 → 电机不转或倒转。每次 PID 输出后必须 CLAMP(0, 1000)
❌ 忘了调弧形 KP → 直线和弧线用同一个 KP → 弧线冲出去。弧线的 DIR_KP 必须是直线的 1.5~2.5 倍
